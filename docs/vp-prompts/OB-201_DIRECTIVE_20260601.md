## §1 — Problem Statement

The platform's authenticated app UI uses the default Next.js / Tailwind styling established incrementally across 200 OBs and 250+ HFs. There is no design system — colors, fonts, spacing, and component styles are scattered as literals across components. A new brand design ("bliss") has been specified with exact tokens (indigo/gold palette, Urbanist/Inter/DM Mono typography, pill CTAs, diamond logo). The architect requires the ability to switch the entire app UI between the current look and the bliss look via a single global toggle, persisted in the database, with zero risk of structural regression on revert.

**This is NOT a defect fix.** It is a presentation-layer capability addition authorized on design/brand grounds at architect discretion. No IAP score is claimed. The toggle's reversibility is the risk control.

**Surface:** Authenticated app UI ONLY. Marketing/public routes are OUT OF SCOPE.
**Scope:** Global platform default (single setting, NOT per-user).
**Persistence:** DB-backed via existing `platform_settings` table (`SCHEMA_REFERENCE_LIVE.md` confirmed: 7 cols, global, no `tenant_id`).
**Mechanism:** CSS-variable token swap under `[data-theme]` root attribute. ONE component tree, TWO token sets. No duplicate/parallel component files. No forked code paths.

Where the app currently uses literal color/font values, replace with CSS variables read from `:root`. Capture the CURRENT values as the `current` token set so the existing look is preserved EXACTLY.

**CRITICAL:** Change NO visual output in this sub-step. The app at `setting="current"` must be pixel-equivalent to the app before this OB.

### §4.2 — Create Both Token Blocks

Create two attribute-scoped token blocks in the app's global CSS:

```css
:root,
[data-theme="current"] {
  --color-primary: <current app's primary color>;
  --color-accent: <current app's accent color>;
  --font-display: <current app's heading font>;
  /* ... every token the app references */
}

[data-theme="bliss"] {
  --color-primary: #2D2F8F;
  --color-accent: #E8A838;
  --font-display: "Urbanist", system-ui, sans-serif;
  /* ... all bliss tokens from §3.2 */
}
```

Wire bliss fonts (`@import` or `@font-face` from Google Fonts, conditional on `[data-theme="bliss"]`), the diamond-texture backdrop, pill radius, shadows, and keyframes so they activate under `[data-theme="bliss"]`.

**Every token a component references MUST exist in BOTH sets.** If any token is defined in one set but not the other — HALT. Report the gap. Do not ship undefined-variable fallbacks.
