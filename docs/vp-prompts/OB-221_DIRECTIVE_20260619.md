## §1 — Problem Statement

The platform currently offers two themes: "current" (dark) and "bliss" (light, Urbanist/Inter/DM Mono). Neither satisfies the brand design language. The dark theme is flat without depth. The bliss theme applies a light palette but lacks the purposeful component vocabulary, layout discipline, and typographic identity that a branded enterprise platform requires.

A comprehensive layout redesign has been designed (by a separate design process) and delivered as a three-file package:

| File | Contents |
|------|----------|
| `vialuce-design-spec.md` | Component anatomy, states, behaviors, accessibility verification, implementation guide |
| `vialuce-ui.css` | Production-ready CSS: tokens + base + all components (306 lines) |
| `vialuce-layout-redesign.html` | Navigable 7-screen prototype — visual source of truth |

This OB introduces **"Vialuce"** as the third theme option. The toggle becomes: Current / Bliss / Vialuce.

**Combined treatment rationale:** The design package is a cohesive system — tokens, typography, layout grid, sidebar, topbar, KPI cards, tables, pills, charts, empty states, and accessibility are interdependent. Splitting into separate OBs would create interim states where partial application produces visual inconsistency worse than either the old or new design.

**Defects resolved:**
- Status badges that look like buttons (blue "Open" → `.pill.open` with icon)
- Persona switcher floating over content → docked in sidebar footer
- 4 inconsistent KPI card variants → one unified `.kpi` component
- Font size/contrast system absent → DM Sans/DM Mono at verified WCAG 2.1 AA ratios
- CTA visual weight absent → `.btn-pri` (indigo) / `.btn-gold` (gold) vocabulary
- Rainbow chart colors → indigo ramp + gold accent discipline

**Step 0b — Register `data-theme="vialuce"` in the toggle mechanism.** The existing dual-theme system uses:

**Step 0c — Three-layer token architecture.** In `globals.css`, add a `[data-theme="vialuce"]` block structured as three declared layers:

```css
/* ═══ VIALUCE THEME ═══ */

/* Layer 0: Primitives — the raw palette. Tenant brand overrides ONLY this layer. */
html[data-theme="vialuce"] {
  --vl-raw-indigo-deep: #2D2F8F;
  --vl-raw-indigo: #4446B8;
  --vl-raw-indigo-light: #6668D8;

  --vl-raw-gold-light: #F5C060;

  --vl-raw-near-black: #1A1A2E;
  --vl-raw-success: #15936A;
  --vl-raw-danger: #DC5454;
  --vl-raw-nav: #1F2154;
}

/* Layer 1: Semantic — role mapping. Maps primitives to purposes. */
html[data-theme="vialuce"] {
  --background: #F4F5FB;
  --foreground: var(--vl-raw-near-black);
  --card: #FFFFFF;
  --card-foreground: var(--vl-raw-near-black);
  --primary: var(--vl-raw-indigo);
  --primary-foreground: #FFFFFF;
  --accent: var(--vl-raw-gold);
  --accent-foreground: #3a2606;
  --muted: #E8EAF3;
  --muted-foreground: #5B637C;
  --border: #E8EAF3;
  --ring: var(--vl-raw-gold);
  /* ... map every existing semantic token the current/bliss themes define */
}

/* Layer 2: Component — slot binding. Components reference these, never Layer 0. */
html[data-theme="vialuce"] {
  --vl-nav-bg: var(--vl-raw-nav);
  --vl-nav-text: rgba(255,255,255,.72);
  --vl-nav-muted: rgba(255,255,255,.42);
  --vl-nav-line: rgba(255,255,255,.09);
  --vl-kpi-accent: var(--vl-raw-indigo);
  --vl-cta-primary: var(--vl-raw-indigo);
  --vl-cta-signal: var(--vl-raw-gold);
  --vl-sidebar-active-bar: var(--vl-raw-gold);
  --vl-status-success: var(--vl-raw-success);
  --vl-status-danger: var(--vl-raw-danger);
  /* ... component slots as needed */
}
```

CC must inventory every semantic token that `[data-theme="current"]` and `[data-theme="bliss"]` define and ensure `[data-theme="vialuce"]` defines the same tokens. Missing tokens produce unstyled surfaces.

**Step 0d — Font loading.** Add DM Sans (300, 400, 500, 700) + DM Mono (400, 500) Google Fonts import, conditional on the vialuce theme. The Bliss theme already demonstrates the pattern for conditional font loading — follow the same mechanism. Under `[data-theme="vialuce"]`:

```css
html[data-theme="vialuce"] {
  --font-sans: 'DM Sans', system-ui, -apple-system, sans-serif;
  --font-mono: 'DM Mono', ui-monospace, monospace;
  font-weight: 300; /* body default */
}
```

**Step 0e — Spacing, radius, elevation, motion tokens.** Add the design system's 8px-base spacing scale, radius tokens, shadow tokens, and motion tokens scoped to `[data-theme="vialuce"]`. Reference: `vialuce-ui.css` lines 55–68.

Commit: `"OB-221 Phase 0: three-layer token architecture + theme registration"`.

**Gate 0:** Toggle to Vialuce in the user menu dropdown → `data-theme="vialuce"` appears on the `<html>` element. Toggle back to Current → Current theme unchanged. Toggle to Bliss → Bliss theme unchanged. Cookie persists across page refresh. Paste evidence of all three states.

### §3.4 — Data Tables + Status Pills + Charts

**Objective:** Under `data-theme="vialuce"`, tables, status indicators, and chart elements follow the brand vocabulary.

**Tables:**
- `.tbl`: header row in DM Mono uppercase, rows with hover, numbers in `.num` (DM Mono, right-aligned)

- Semantic numbers: `.num.up` (green), `.num.down` (red)

**Status pills (not buttons):**
- `.pill.success` (green): Completed — includes icon
- `.pill.danger` (red): Failed — includes icon
- `.pill.open` (indigo): Open — includes icon
- `.pill.neutral` (gray border): Monthly, Biweekly, etc.
- These are informational indicators, not interactive elements. No cursor:pointer. No click handler.

**Charts:**
- Mandatory palette: `--vl-raw-indigo-deep → --vl-raw-indigo → --vl-raw-indigo-light → #9A9CE0`, with `--vl-raw-gold` as accent
- Performance tiers semantic: Exceeding=success, Meeting=indigo, Approaching=gold, Missing=danger
- Line chart convention: Actual = solid indigo, Budget = dashed gold
- Horizontal bars (`.hbars/.hbar`), donut and line as inline SVG

**Row actions:** `.gbtn` (ghost button) + `.iact.del` (trash, red on hover). Right-aligned (`.row-act`).

Commit: `"OB-221 Phase 4: tables + pills + charts under vialuce theme"`.
