## §1 — Problem Statement

HF-305 and HF-306 built the palette remap and applied design-language elements. The result is partially there but wrong in material ways because the bliss token values in `globals.css` were derived from an earlier hex-based `index.css` that is NOT the production source.

The architect has now supplied the **authoritative** production stylesheet (`styles.css` from the Lovable marketing repo). It uses OKLCH throughout, and several values differ materially from what's currently in the bliss token block:

| Token | Current in globals.css (wrong) | Authoritative (correct) | What it means |
|-------|-------------------------------|------------------------|---------------|
| `--ink` / `--foreground` | `#0F0A1E` (near-black) | `oklch(0.24 0.11 274)` | Text is deep indigo-tinted, NOT black |
| `--background` | `#F8F8FF` (offwhite) | `oklch(1 0 0)` | Page bg is pure white, NOT offwhite |
| `--gold-deep` | `#B07A15` | `oklch(0.58 0.16 65)` | Eyebrow label color |
| `--muted-foreground` | `#666080` | `oklch(0.52 0.06 274)` | Secondary text has indigo cast |
| `--border` / `--input` | `#E4E4EE` | `oklch(0.92 0.01 274)` | Borders are indigo-tinted, not neutral |
| `--indigo-soft` | `#F0F0FA` | `oklch(0.97 0.025 274)` | Slightly different |
| `--gold-soft` | `#FAF6E8` | `oklch(0.97 0.05 86)` | Different |
| `--card` | offwhite | `oklch(1 0 0)` | White, not offwhite |
| `--muted2` | missing | `oklch(0.86 0.012 274)` | New token, missing entirely |

Additionally, the font body token in the authoritative source is `--font-sans`, not `--font-body`. The `.eyebrow` base color is **indigo**, not gold-deep (`.eyebrow-gold` is a variant). These naming/value mismatches explain why fonts aren't rendering correctly and labels look wrong.

The authoritative source also defines a complete design vocabulary (utility classes) that the app doesn't have: `.eyebrow`, `.hairline`, `.lift`, `.reveal`/`.reveal-show`, `.tint-a/b/c/d`, `.shimmer`, `.underline-wipe`, plus all the animation keyframes. These are what make the marketing site feel like the marketing site.

**Beyond tokens:** the Manager view and Reconcile view still have dark gradient backgrounds and dark card surfaces. The diamond logo is not rendering. Multiple pages have not been reached by HF-305/306's remediation.

This HF replaces the bliss token values with the authoritative OKLCH values, adds the missing design vocabulary, fixes remaining dark surfaces, and delivers the diamond logo.
