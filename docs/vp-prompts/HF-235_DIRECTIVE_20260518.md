# HF-235: PASS 4 PROMPT — REMOVE NON-DETERMINISTIC SAMPLE ROWS

Predecessor: HF-234 (PR #412 separation of concerns), DIAG-049 (PR #413
post-HF-234 convergence state extraction).

Defect: Pass 4 (`generateAISemanticDerivations`) fetches 3 rows from
`committed_data` with `.limit(3)` and no `data_type` filter / no
ordering, then injects them as JSON into the AI prompt under a "Data
sample" header. Non-deterministic: rows returned may belong to a
different data_type than the metrics reference (roster / quota /
transaction shapes). When the sample happens to include rows with the
relevant categorical value, the AI emits a filtered derivation; when
it doesn't, the AI emits a gap or filter-less derivation. Column
descriptions from `capabilities` (built at the same prompt site from
`cap.numericFields` / `cap.categoricalFields` / `cap.booleanFields`)
already carry every distinct value the AI needs for filter selection.

Fix surface (one file, ~15 lines removed):
- `web/src/lib/intelligence/convergence-service.ts:generateAISemanticDerivations`:
  delete the sample query block (lines 2650-2663) and the
  `${sampleData}` injection in the user prompt template (lines
  2748-2749). Column descriptions are sufficient for filter derivation.

per standing rule 5.
