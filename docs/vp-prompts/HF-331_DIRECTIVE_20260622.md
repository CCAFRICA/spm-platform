## §1 — PROBLEM STATEMENT

### Defect

OB-178 Phase C migrated auth state resolution from client-side cookie reading (`getSession()`) to server-side verification (`getUser()`) per Decision 142 and DS-019 §4.3. The migration was incomplete. At least one `getSession()` callsite survives in the server render path. This is confirmed — not hypothesized — by a Vercel server-side log entry:

```
2026-06-22 04:27:30.008 [info] [users] / status=200
2026-06-22 04:27:30.275 [warning] Using the user object as returned from supabase.auth.getSession()
or from some supabase.auth.onAuthStateChange() events could be insecure! This value comes directly
from the storage medium (usually cookies on the server) and may not be authentic. Use
supabase.auth.getUser() instead which authenticates the data by contacting the Supabase Auth server.
```

This warning fired server-side on the `/` route at 04:27:30 UTC, three seconds after a production `auth.shell.hydration_timeout` event on `/select-tenant` for a platform-scope user (eoadmin@vialuce.com, tenant_id null, role platform, single profile row).

### Symptoms (two, one root cause)

**Symptom A — Desktop hydration stall (`/select-tenant`).** Platform-scope sessions (tenant_id = null) intermittently stall on `/select-tenant`. The auth shell hydration gate times out. The user sees a spinner, re-logs in. `auth.shell.hydration_timeout` events in `platform_events` confirm. Intermittent because it is a race: if the cookie updates before the `getSession()` read, it works; if not, the stale token (pre-MFA, aal1) causes the shell to never resolve `isLoading`.

**Symptom B — Mobile MFA non-completion (iOS / Firefox-iOS).** Platform-scope user (platform@vialuce.com) logs in on iPhone (FxiOS/152.0). Switches to authenticator app for TOTP code. iOS suspends the backgrounded WKWebView tab. On return, the MFA verify page re-renders or re-checks auth state through the surviving `getSession()` path, reads the stale pre-MFA cookie, and either does nothing or silently fails. `platform_events` trail shows `login.success` at 18:49:42 with no subsequent `mfa.verify.success` or `mfa.verify.failure` — the verify handler never fired. Contrast: the same user on desktop Mac/Chrome completes MFA every time (13 consecutive successes in the 24-hour window).

### Defect class

**Incomplete class closure of OB-178 Phase C.** The named callsites (`fetchCurrentProfile`, `getServerAuthState`, middleware `getUser()`) were migrated. At least one adjacent-arm consumer was missed (DD-2: class-closure HFs enumerate every compensation-pattern consumer at drafting time). The surviving callsite is the root; the two symptoms are downstream consequences. Closing only one symptom (e.g., adding a `visibilitychange` handler for mobile) without retiring the root would be instance closure masquerading as class closure (AP-D2).

## §2 — SUBSTRATE-BOUND DISCIPLINE APPLICATIONS

**Decision 142 (httpOnly Cookie Architecture):** server-side session resolution. Client components receive auth state as props. Browser client used ONLY for login/logout write operations. `getSession()` reads cookies without server verification. `getUser()` round-trips to Supabase Auth server. Every auth-state read in a server component, layout, middleware, or API route MUST use `getUser()`. This HF completes that contract.

**Decision 140 (Supabase Auth Confirmed):** Supabase Auth is the provider. The fix uses Supabase's own `getUser()` API. No external provider introduced.

**SR-34 (No Bypass):** the fix retires the `getSession()` class across the entire codebase, not just the instance that fired the Vercel warning. Every `getSession()` consumer in a server-side auth-gating context is in scope.

**SR-39 (Compliance Verification Gate):** `getSession()` in a server-side auth path violates OWASP session management (token not verified with server) and SOC 2 CC6 (session validation must be server-enforced). The fix restores compliance.

**Korean Test (T1-E910 v2):** not directly exercised — no field identification or structural heuristic involved. Noted for completeness.

**Reconciliation-channel separation:** no ground-truth values present. Not exercised.

**Classify each hit into one of three dispositions:**

**Deliverable:** committed inventory file at `docs/diagnostics/HF-331_GETSESSION_INVENTORY.md` with pasted grep output, per-hit classification table, and the count of REPLACE-classified hits.

### OBJECTIVE 2: Retire All REPLACE-Classified `getSession()` Consumers

For every REPLACE-classified callsite from Objective 1:

**The migration pattern** (already proven in `fetchCurrentProfile` and `getServerAuthState`):

```typescript
// BEFORE (defective — reads from cookie, not server-verified)
const { data: { session } } = await supabase.auth.getSession();
const user = session?.user;

// AFTER (correct — round-trips to Supabase Auth server)
const { data: { user } } = await supabase.auth.getUser();
```

For `onAuthStateChange` listeners that read the event's `session.user` for auth-gating:

```typescript
// BEFORE (defective — event payload is cookie-sourced)
supabase.auth.onAuthStateChange((event, session) => {
  const user = session?.user;
  // ... auth-gating logic using user
});

// AFTER (correct — verify with server on state change)
supabase.auth.onAuthStateChange(async (event, session) => {
  if (event === 'SIGNED_IN' || event === 'TOKEN_REFRESHED' || event === 'MFA_CHALLENGE_VERIFIED') {
    const { data: { user } } = await supabase.auth.getUser();
    // ... auth-gating logic using server-verified user
  }
});
```

**Constraints on the migration:**
- Preserve pre-HF behavior exactly (DD-7). The downstream logic (role resolution, redirect routing, tenant resolution, hydration gating, MFA enforcement) is NOT modified. Only the auth-state source changes.
- Do not introduce new behavioral branches, new event handlers, or new UI elements.
- Do not modify RETAIN-classified callsites.
- If a callsite is in a shared utility imported by both server and client contexts, ensure the migration does not break client-side login/logout write paths. If architectural ambiguity exists, HALT-1.
