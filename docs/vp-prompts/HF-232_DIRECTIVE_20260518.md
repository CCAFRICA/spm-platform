# HF-232: HC DECISION TREE — reference_key DISCRIMINATES TRANSACTION FROM TARGET

Predecessor: HF-230 (PR #408 primitive-based decision tree).

Defect: HF-230 Branches 3 and 4 distinguish target from transaction by
`identifierCount` (1 vs ≥ 2). CRP sales files carry
`transaction_id:identifier@0.95` + `sales_rep_id:reference_key@0.95`, so
`identifierCount === 1` and Branch 3 fires → misclassified as `target`.

Fix surface (one file, ~5 lines):
- `web/src/lib/sci/hc-pattern-classifier.ts` — Branches 3 and 4 now
  discriminate on `hasReferenceKey`:
    `identifierCount >= 1 && hasReferenceKey`  → transaction
    `identifierCount >= 1 && !hasReferenceKey` → target
  Branch 5 (`identifierCount === 0` → reference) unchanged.
