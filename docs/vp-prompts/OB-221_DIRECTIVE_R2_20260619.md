## §1 — Problem Statement

The platform currently offers two themes: "current" (dark) and "bliss" (light, Urbanist/Inter/DM Mono). Neither satisfies the brand design language. The dark theme is flat without depth. The bliss theme applies a light palette but lacks the purposeful component vocabulary, layout discipline, and typographic identity that a branded enterprise platform requires.

A comprehensive layout redesign has been designed (by a separate design process) and delivered as a three-file package:

| File | Contents |
|------|----------|
| `vialuce-design-spec.md` | Component anatomy, states, behaviors, accessibility verification, implementation guide |
| `vialuce-ui.css` | Production-ready CSS: tokens + base + all components (306 lines) |
| `vialuce-layout-redesign.html` | Navigable 7-screen prototype — visual source of truth |

This OB introduces **"Vialuce"** as the third theme option. The toggle becomes: Current / Bliss / Vialuce.

**Combined treatment rationale:** The design package is a cohesive system — tokens, typography, layout grid, sidebar, topbar, KPI cards, tables, pills, charts, empty states, and accessibility are interdependent. Splitting into separate OBs would create interim states where partial application produces visual inconsistency worse than either the old or new design.

**Defects resolved:**
- Status badges that look like buttons (blue "Open" → `.pill.open` with icon)
- Persona switcher floating over content → docked in sidebar footer
- 4 inconsistent KPI card variants → one unified `.kpi` component
- Font size/contrast system absent → DM Sans/DM Mono at verified WCAG 2.1 AA ratios
- CTA visual weight absent → `.btn-pri` (indigo) / `.btn-gold` (gold) vocabulary
- Rainbow chart colors → indigo ramp + gold accent discipline
