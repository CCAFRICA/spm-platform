## §1 — Problem Statement

Three defects block the MIR tenant onboarding flow. All are platform defects, not MIR-specific.

### Defect A — /data/import crashes

Navigating to `app.vialuce.ai/data/import` renders "Something went wrong. A critical error occurred. Please refresh the page." The page is unreachable. This blocks all data upload for any tenant.

**Evidence:** Screenshot `CLT227-HF329-Import_Upload_Data_Erro.pdf`. The Lifecycle Cockpit's "Upload Data →" button (HF-326 onboarding checklist) routes here and hits the crash.

**CC action:** Read the `/data/import` page component. Check Vercel function logs for the error. Identify the crash (null reference, missing context, component error). Fix structurally.

### Defect B — Period auto-creation detects 3 of 6 months

MIR has 6 months of transactional data (January through June 2025). The auto-detect endpoint created only 3 periods: January 2025, April 2025, May 2025. February, March, and June are missing.

```
Created 3 periods for tenant 972c8eb0: January 2025, April 2025, May 2025
```

The detect endpoint scans `committed_data.source_date` values, groups them by month, and suggests periods. Three months of data are not being detected. Possible causes: source_date values for Feb/Mar/Jun are null, malformed, or outside the scan window; the grouping logic skips months; the "already exists" filter incorrectly excludes them. CC reads the detect endpoint code AND queries the actual `committed_data.source_date` distribution for tenant `972c8eb0-e3ae-4e4c-ad30-8b34804c893a` to determine which.

### Defect C — Calculate UX insufficient

The plan selector dropdown (HF-326) shows plan names only. For a tenant with multiple plans (MIR has 5 interpreted from PDFs), this is insufficient:

**C1 — No plan intelligence.** The dropdown shows "Plan de Bono por Cartera Nueva" with no context — how many components, what entity count, whether it has been calculated for this period. The user cannot make an informed selection.

**C2 — No batch calculation.** Each plan must be selected and calculated individually. For 5 plans × 6 periods = 30 calculation runs, this is operationally unacceptable. The cockpit needs a "Calculate All Plans" action for the selected period, and ideally a "Calculate All Periods" action.

**C3 — Plan status per period.** The period ribbon shows "No calc" but doesn't indicate which plans have been calculated and which haven't. A period with 3 of 5 plans calculated should show partial status.

**C4 — VIALUCE THEME ONLY.** Changes render under `useIsVialuce()`.
