## §2 — Substrate-Bound Discipline Applications

- **Decision 158 (consumer-side mirror — strict).** The LLM **recognizes**: given a surface's free-form purpose text and the tenant's free-form `comprehension_artifacts` characterizations, it returns which field(s) satisfy the purpose, with confidence. Recognition is **free-form on both sides** — the purpose is natural language, the characterization is natural language; the LLM bridges them. Code **never** constrains the LLM's output into a property schema and **never** substring-matches the characterization text (that is the Obj-4 fallback class — C3). Code **constructs**: persists the recognized binding, emits the signal, renders deterministically.
- **No Fixed Taxonomy / Korean Test (the registry bright line).** HF-337 introduces **no enumerated set** of purposes, metric roles, fields, or structural properties to gate resolution. The binding store is keyed on **`structural_fingerprint_hash` + `surface_id`** — a hash computed from the data's own shape, plus a fixed *product* surface (surfaces are product features, finite by product design; a new *domain* adds a fingerprint, never a surface). It grows when the **system recognizes a new shape** (by encounter), never when a **developer edits an entry** (by maintenance). *A store we are innovating away from grows when a person edits it; this store grows when the system recognizes something new. The first is taxonomy; the second is learning.* If the only way to make resolution work would require a developer-authored intent/role/field key → **HALT-REGISTRY**.
- **Strict-(2) foundation (meaning originates in comprehension).** The data-intelligence foundation is pure comprehension-driven salience: a new domain imports, comprehension characterizes its fields in their own terms, surfaces can render that. A surface's opinionated purpose is a **lens over comprehended salience** — it sharpens to the recognized binding if one resolves, and **falls back to comprehension-driven salience if none does**. The lens is a **render preference that degrades gracefully**, never a lookup that must succeed. This is what lets domain N+1 arrive with **zero foundation change**: the surface holds no presumption the new domain must satisfy.
- **Expression learns, too (Role 4 at the expression layer — the new scope).** The binding recognition is itself a learning event: it emits a signal and accumulates a fingerprint→binding artifact. Same-tenant re-encounter reads it (Progressive Performance at tenant scope — recognize once, read forever). **Cross-tenant** transfer (a new tenant whose comprehension shape matches a known fingerprint inheriting the binding at cold-start) is **OB-235's learner-core** — but the **emission and the fingerprint-keyed store are in scope here**, shaped for OB-235 to read without reconciliation.
- **T1-E952 (Adjacent-Arm Drift).** P0 sweeps **every** consumer of the eradicated keys; the class is closed, not the `network_pulse` instance.
- **C2 (fail-loud).** Salvage-partial is made logged/structured (P1b); the recognizer returns a **structured-unresolved** on no match → graceful comprehension-driven fallback, never a silent blank (P2c).
- **C6 (calc immutable).** No foundation fix and no binding code touches the calc engine reader, `convergence-service`, `lib/calculation/`, or `committed_data.metadata.field_identities`. Additive only.
- **SR-2 / Progressive Performance.** Recognition is one cached call per (tenant fingerprint × surface) not-yet-seen; re-encounter is a deterministic read. No per-render, no per-entity AI call.
- **Reconciliation-channel separation.** BCL ($312,033) and MIR (S/6,091,073.68) ground truth are architect-channel. CC reports calculated values **verbatim** and never reconciles against a reference. Any ground-truth figure found committed in the repo is flagged for architect scrub (P0), not deleted.
- **T1-E905 (Prove Don't Describe).** Every gate carries pasted evidence (code/terminal/grep/screenshot). Self-attestation rejected.
