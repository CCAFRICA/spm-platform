## §1 — Problem Statement

**Objective of this directive: Casa Diaz imports, end-to-end.** Tenant `2d9979ba-5032-48a7-bccf-1928f3e6dadf`. Two deliverables define success: (1) the 8-sheet banded commission workbook `COMISIONES___AUTORIZADOS_-_copia.xlsx` imports through the browser and produces its complete, calculable compensation plans, visible with their components in Plans & Canvas; (2) the large JDE extract (86K×87, ~42MB) imports and commits its transaction rows. Achieving this requires the import process itself — broken for days across a chain of interacting defects — to work.

The defect chain, each item observed live in the current session (not inherited from reports):

**D1 — Recognition corruption: coarse atom-fingerprint collisions.** The shared column-recognition memory keys columns by a fingerprint that captures value-shape but not identity. Live evidence: the plan column `Componente` and the employee-name column `Nombre_Completo` share hash `bf2acb98cc11`; `Componente` was claimed as "a free-text human name string… practically unique per employee" @0.99. Downstream: a plan sheet whose second column reads as "people's names" classifies as a roster, is excluded from plan interpretation, and the plan is built from a fraction of its sheets (observed: interpretation ran on 1 sheet / 329 chars; the component-definition and monthly-targets sheets never reached it; 2 components produced where the anchor structure had 4). Multiple distinct measure columns also collide (`Nivel` ≡ `Meta Colocación ($)` ≡ `Meta Depósitos ($)` at one hash), driving recurring role-AMBIGUOUS churn. This is the reconciliation- and Casa-Diaz-blocking defect: an 8-sheet banded workbook is maximally exposed to cross-sheet collisions.

**D2 — Construction in the wrong place: LLM-generated rate tables.** Rate-table components are constructed by the model emitting the entire calculation graph cell-by-cell (25-cell matrices at 40-52 s each, observed repeatedly), with a truncation defect class (max-tokens raises; `insight.partial_salvage`-style cutoffs) and run-to-run nondeterminism (2 vs 4 components across runs of the same file). Decision 158 assigns construction to deterministic code: the model recognizes ("this is a rate matrix: compliance bands × quality bands, one per role variant; these are the cell values' locations"), and code constructs the cells from the already-parsed, de-banded grid. Milliseconds, exact, identical every run, truncation class eliminated.

**D3 — Registries reintroduced.** HF-367 removed the keyword-scan classifier but introduced `MEASURE_NATURE` / `TEMPORAL_NATURE` word lists (acknowledged at the time as scope growth). Any developer-curated vocabulary participating in classification violates the subtraction arc and constrains LLM expression: the model must express recognition in its own structured fields (`identifies`, `data_nature`, `scope_role`, `nature_role`), read directly — not filtered through an enumerable list a developer can "complete."

**D4 — No truthful job status; controls absent where the work happens.** The import screen shows a perpetual spinner/counter that does not track reality (observed: "0 of 4, 183 s elapsed" while the server had committed at 65 s and finalize was done at 66 s). The fleet Ingestion view shows every job green "classified" including unfinished ones, dead 0.0% metrics beside a populated queue, a self-contradictory "No ingestion events yet" panel, no stuck detection, no usable kill, and a dark off-brand theme. There is no single truthful source of job state, and no stop/kill control on the import screen itself.

**D5 — The insight step blocks completion and half-fails.** Post-commit insight generation runs ~74 s on the critical path and truncates (`insight.partial_salvage: whole-array parse failed; recovered 7…`). Completion must gate on committed data, not on this step.

**D6 — Large-file admission.** The storage bucket's `file_size_limit` is unset, falling back to a floor below the ~42MB JDE file; upload of the second Casa Diaz deliverable is blocked at the door. The bucket change is an architect action (SR-44); the code side must fail loud with the actual limit when an upload exceeds it, and the oversized (windowed-parse) path must be verified end-to-end.

**D7 — Pathway fragmentation.** The import arc has accumulated parallel paths (sync vs async worker vs streamed/windowed; multiple finalize triggers now coalesced by claim; classify-time vs commit-time recognition surfaces). AP-17 requires one code path per function; every divergence is a place where a fix lands on one path and not the other — the recurring mechanism by which "fixed" defects resurfaced this week.

Combined-treatment rationale: D1-D3 corrupt what the import produces; D4-D5 make it impossible to see or control; D6-D7 gate the Casa Diaz deliverables specifically. No subset unblocks the objective.

## §2 — Substrate-Bound Discipline Applications

**Decision 158, both halves, everywhere.** The model recognizes; deterministic code constructs and guarantees. D1's fix strengthens recognition integrity (identity-keyed memory so recognitions cannot cross-contaminate). D2's fix moves construction where the law puts it (code builds the rate cells from parsed data; the model's output is recognition of the matrix's structure and semantics, not the generated graph). No phase may keyword-scan model prose, re-classify to overrule the model, or default from a list.

**Subtraction arc / NO REGISTRIES with LLM expression preservation.** D3 is a removal, not a replacement: the word lists are deleted and the consuming code reads the model's own structured recognition fields. The Validation Premise Law's acceptance test applies to every touched surface: if a developer could make a check "more complete" by editing a list, it is a registry — reject. The model's free-form expression is preserved end-to-end; structured fields are the only contract.

**Korean Test (AP-25/E910).** The fingerprint fix, the rate-table constructor, and the row-ordinal guard carry zero language-specific string literals. Identity keying uses the column's header and structural shape as data, not as matched vocabulary.

**AP-17 single code path.** D7's unification is bounded (§3.7): one recognition surface, one commit surface, one finalize dispatch, with the streamed/windowed variant as a size-gated branch of the same path — not a second implementation.

**Sequence-independence and determinism.** The import result is identical regardless of file order and identical across repeated runs. The Phase E gates assert both.

**Progressive Performance preserved.** Fixes must not destroy warm-path acceleration: identity-keyed atoms still accelerate true re-encounters; deterministic construction makes the expensive step cheap for everyone. Phase E includes a second-encounter gate.

**Reconciliation-Channel Separation.** No verification anchor values appear in this directive. CC reports produced totals and plan structures verbatim; the architect performs all reconciliation and all Plans & Canvas visual confirmation (SR-44).
