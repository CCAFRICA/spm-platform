## §1 — Objective

The import flow (`/operate/import`) has two states — Analyze and Complete — both with wrong information hierarchy and internal jargon. Under Vialuce, redesign both screens to put the platform's intelligence front and center, replace developer terminology with user language, and connect the workflow to the Lifecycle Cockpit.

---

## §2 — Import Analyze (the platform is learning)

The admin uploaded files. They know what they uploaded. What they need to know is: **did the platform understand my data?**

**Current state (wrong):** Files listed prominently at top. Intelligence metrics (comprehension, atoms, fingerprints) buried in a gray footer at the bottom.

**Target state:** Invert the hierarchy.

**TOP — Intelligence Summary** in `.kpi` cards (the headline):
- **Sheets Comprehended** — X of Y with progress indicator. "Understanding your data..." becomes a live comprehension status.
- **Fields Recognized** — count of known patterns matched (replaces "Atoms recalled")
- **New Patterns Learned** — count of novel patterns discovered (replaces "Atoms learned (novel)")
- **Data Signatures** — fingerprints stored (replaces "Fingerprints recognized")

**BELOW — File List** as reference detail. The uploaded files with their comprehension status pills. This is secondary — the admin already knows what they uploaded.

**Gray footer eliminated.** All intelligence content lives in proper `.card` surfaces with design spec vocabulary.

**Vialuce-scoped:** `useIsVialuce()` conditional. Dark/Bliss render unchanged.

---

## §3 — Import Complete (the platform finished learning)

The admin needs to know: **what was imported, and what do I do next?**

**Current state (wrong):** "595 records imported" with confusing metrics (BINDINGS, ATOMS, FINGERPRINTS, PULSES) and CTAs using internal terminology ("Review Bindings →").

**Target state:**

**Hero — Import Summary** in `.kpi` cards:
- **Records Imported** (595) — DM Mono, large
- **Entities Found** (85)
- **Content Units** (2)

**Intelligence Summary** — What the platform learned, in user language. `.card` with clean rows:

| Internal term | User-facing label |
|---|---|
| Memory Saved / Atoms recalled | Recognized Patterns |
| Atoms learned (novel) | New Patterns Learned |
| Bindings injected | Field Mappings Applied |
| LLM calls made | AI Analysis Steps |
| Fingerprints stored | Data Signatures Stored |
| Signals captured | Quality Signals Captured |
| Rows committed | Records Committed |
| Pulses | Remove entirely OR "Processing Events" |

**Carrier Intelligence** — the data quality assessment (89.2% — Review recommended). This is important: render as `.insight` banner with gold accent. The quality percentage and recommendation are the signal.

**CTAs — simplified and prioritized:**
- **PRIMARY (`.btn-gold`):** "Go to Calculate →" — navigates to `/operate` (the Lifecycle Cockpit from OB-226). This is the natural next step.
- **SECONDARY (`.btn-pri`):** "Review Data Quality →" — replaces "Review Bindings" (user language). Links to wherever data quality review happens.
- **TERTIARY (ghost):** "Import More Data" and "View in Intelligence →" (replaces "View in Stream")

**Session detail table** — the imported units with types and row counts. `.card.flush` + `.tbl` with DM Mono. Reference detail, not the headline.

**Vialuce-scoped:** `useIsVialuce()` conditional. Dark/Bliss render unchanged.
