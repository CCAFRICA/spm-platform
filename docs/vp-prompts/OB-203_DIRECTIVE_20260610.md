## §1 — Problem Statement

Production evidence 2026-06-10 (Brasa y Maíz, datos-cadena-restaurantes-mx.xlsx, 16 sheets, 162,956 rows, three identical failures): one in-line header-comprehension LLM call covering all novel sheets ran 68s, returned unparseable JSON (`[SCI] Header comprehension JSON parse failed (AIService fallback)`), fell back silently to heuristics (`llmCalled=false avgConf=0.00 cols=0 insights=0`), misclassified the employee roster as transaction@76%, and produced a 16-unit proposal presented as comprehended. HF-247's quality gate then blocked all fingerprint writes (`columnRoles contains 'unknown'` ×13), guaranteeing identical cold-start failure on every future attempt. Four stored pre-gate fingerprints carry undefined roles and Tier-1 match forever, injecting nothing useful.

The structural classes, substrate-confirmed via IRA invocations ecc3824e and the DS-027 scope-coherence gate: comprehension gates ingestion (T1-E902 inversion), session state is ephemeral (T1-E903 chain unmodelable), recognition is sheet-opaque (T1-E906 read-before-derive unreachable), failure is silent (T1-E910 violation). Fragmented instance-level fixes are explicitly rejected (E907; architect directive 2026-06-10: comprehensive solution). This OB closes the class at all four dimensions.

**Behavior preservation:** the calculation engine, plan interpretation, and all post-proposal pipelines are untouched. **Blast radius statement:** Phases 0-6 never touch the closed proof tenants — all witness imports use sandbox/fixture tenants only. Phase 7 then deliberately regenerates the proof tenants through the new pipeline as the arc-closing regression (architect-gated clean-slate per SR-44, baselines preserved first, rule_sets retained, anchors must reproduce exactly). The live exposure is future imports through the SCI proposal path, gated by Phase 0 consumer enumeration, the deterministic failure-signature predicate (HALT-7), the byte-identical success-path test, and the Phase 2 early regression witness (EPG-2.4). Named expansions (DD-7): atom-level recognition, durable unit states, failure surfacing, interaction-signal capture, revised import dialogs. Regression witness: Meridian Q1 totals (185,063 / 175,585 / 196,337) must remain exact post-arc (Phase 6).

## §2 — Substrate-Bound Discipline Applications

- **DS-027 DI-1:** no unit's persistence may depend on any unit's comprehension. Every phase's EPG includes the DIAG-050 check: no staging/commit path gains a comprehension-state condition.
- **DS-027 DI-2 / Decision 158:** LLM payload bounded by novel atoms + sheet context, schema-validated compact response, one repair retry; LLM recognizes, code constructs. Never O(rows), never O(known structure).
- **DS-027 DI-3 / DI-9 / DI-10 / T1-E910:** atom fingerprints are structural (value distributions, types, cardinality, repeat ratios, bucketed), carry construction-algorithm version, and exclude tenant-identifying values by construction. Header text is display metadata, never identity. Verbatim fingerprint-construction function pasted for architect inspection (EPG every phase that touches it).
- **DS-027 DI-4:** every comprehension boundary — including workbook-graph synthesis as a unit — produces a named, durable, user-visible failure state. No silent fallback path may survive this OB.
- **DS-027 DI-5:** signals always written; Synaptic Density governs read/activation only. Any write-time gating is a violation (HALT-3).
- **DS-027 DI-6 / Decision 64 v2 / G7:** one canonical surface. All machine and interaction signals route through the existing `classification-signal-service.ts` conventions on `classification_signals`.
- **DS-027 DI-7:** every blocked or failed learning write emits a remediation signal.
- **DS-027 DI-8 / Decision 92 extension:** atom identity persists across periods without re-derivation.
- **Blind-holdout / structural-analog discipline (Decision 154 operationalized; architect directive 2026-06-10):** real tenant data files are NEVER committed to the repo and never supplied to CC. CC develops and tests against structural-analog fixtures it generates (seeded generator, committed as code); the real witness file is architect-held and architect-executed, blind to the implementation. Any test or witness that appears to require committing a tenant data file is HALT-10.
- **Vertical Slice Rule:** every phase ships engine + experience together. No engine-only or UI-only PR.
- **Decision 73 (Proposal + Confirmation) preserved:** the observer experience proposes; the user confirms or corrects; every correction is a signal.

## §3.2 — Phase 2: Hierarchical Recognition & Comprehension Decomposition (R1)

Engine:
- **Atom fingerprints:** per-column structural fingerprint (bucketed distributions/types/cardinality/repeat/pattern flags per the existing composite-signature mechanics extended to column scope), carrying `algorithm_version` (DI-9) and constructed to exclude raw values and tenant identifiers (DI-10). Fingerprint store extended by migration per the SQL Verification Gate; migration enumerates a bridge view or version column so any future algorithm change keeps prior signal reachable.
- **Composition identity:** sheet identity = ordered multiset of atom fingerprints + structural profile; Tier-1/2 matching operates per atom and per composition; partial recognition is native — known atoms carry accumulated roles regardless of novel neighbors.
- **Read-before-derive gate:** no LLM dispatch for any atom with prior signal at sufficient confidence; paste the gate function verbatim.
- **Decomposed comprehension:** per-unit bounded LLM calls on novel residue only, with sheet context; compact schema-validated response; on parse/validation failure one repair retry feeding the validation error, then `failed_interpretation` (Phase 1 surface). Implemented at the AIService entry point so all consumers inherit validation.
- **Flywheel accumulation:** atom-level signals at tenant scope; foundational/vertical scope writes carry DI-10-compliant fingerprints only.
Experience: per-sheet recognition provenance in the proposal — tier, fraction of atoms recognized vs novel, comprehension cost indicator. The Empleados-class misclassification surface shrinks here; full relational correction is Phase 6.
**Structural-analog generator (this phase, committed as code):** a deterministic seeded generator producing workbooks of the witness's structural class — cover sheet, entity roster, reference tables, high-cardinality fact grain, derived aggregations — with randomized-token or non-Latin vocabulary, never real tenant vocabulary. All Phase 2-6 tests run against generated analogs.
Tests (against analogs): novel-residue isolation (28 known + 2 novel atoms → comprehension payload covers 2); repair-retry path; version-bridge readability; DI-10 fingerprint content assertion (no raw values); cross-vocabulary generality (same structural class, different token language → equivalent recognition).
EPGs: EPG-2.1 Korean Test — fingerprint construction + atom matching function bodies pasted verbatim, structural literals enumerated, zero language/domain strings; EPG-2.2 payload-bound proof — pasted comprehension request for a generated structural analog showing O(novel atoms); EPG-2.3 DIAG-050; EPG-2.4 proof-tenant regression witness (early, per architect risk directive 2026-06-10) — re-import the Meridian source file into a SANDBOX tenant (never the closed Meridian tenant): bindings must be token-complete and classifications consistent with the rule_set cac8c391 expectations; paste the binding verification output. Phase 2 does not merge without EPG-2.4. PR: `OB-203-phase-2`.

## §3.3 — Phase 3: Durable Comprehension State (R2)

Engine: unit states (`persisted → profiled → recognized(tier) → comprehended → classified → bound`; `failed_interpretation`, `resolved`; synthesis-as-unit reserved for Phase 6) written as signals on `classification_signals` with a durable import-session identity grouping them. `persisted` unconditional (DI-1). The in-memory SynapticIngestionState becomes a working cache rebuilt/resumable from the signal surface; retry of a failed unit re-runs comprehension without re-ingestion. Human resolution writes a correction signal with `source: user_corrected` provenance via the existing override mechanics generalized to every state.
Experience: import surface reads states live (poll or subscribe per existing app conventions) — sheets appear at `persisted` and progress visibly; a failed unit is retryable from the UI (retry action lands here as the minimal resolution affordance; the full action set is Phase 5).
Tests: state monotonicity per unit; independence across units; session resume after process restart; retry-without-reingestion; resolution provenance.
EPGs: EPG-3.1 G7 — zero new signal tables/channels; EPG-3.2 DI-1 — persistence path free of comprehension conditions; EPG-3.3 read-before-derive consumes durable signal (query evidence). PR: `OB-203-phase-3`.
