# HF-230: HC PATTERN CLASSIFIER — PRIMITIVE-BASED DECISION TREE

Predecessor: HF-105 (PR #207 two-level model), HF-229 (PR #407 Decision 108
enforcement at the entity_roster pattern).

Defect class: the Level-1 HC pattern classifier was a registry of four
enumerated file shapes (entity_roster, repeated_measures_over_time,
lookup_table, per_entity_benchmarks). Every novel shape needed a new
pattern. The classifier never asked first-principles "what does the LLM's
role composition tell me" — it asked "does this file's structural
fingerprint match one of four known shapes."

Single surface (one file, ~60 lines replaced):
- `web/src/lib/sci/hc-pattern-classifier.ts:classifyByHCPattern` — replace
  the four enumerated pattern blocks with a decision tree derived from
  three HC role primitives: `reference_key+identifier presence`, `measure
  presence` (including `currency`), and `identifier count` (0 / 1 / 2+).

Resulting branches: `dimensional_lookup` → reference, `entity_definition`
→ entity, `entity_targets` → target, `event_transactions` →
transaction. Level-2 CRR Bayesian fallback unchanged (still runs when
the tree returns null — minimum-coverage gate at 50% column roles
≥ 0.80 confidence).
