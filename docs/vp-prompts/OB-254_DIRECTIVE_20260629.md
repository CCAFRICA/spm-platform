## §2 disposition record — D1, D2, D3 (architect, 2026-06-29, post-HALT-1)

Phase 1's surface map (`docs/diagnostics/OB-254_PHASE1_SURFACE_MAP.md`, EPG-1) fired HALT-1: the originally-drafted insertion point ("first step inside Content Profile, on the raw grid, before the fingerprint") does not exist in the live topology. The fingerprint precedes Content Profile (`process-job:186/197` before `:232`); parse collapses the grid to header-keyed row objects immediately (`sheet_to_json:158-160` / `openSheetWindow:151`); and OB-251 windows over-ceiling sheets on a bounded sample. Dispositions:

**D1 — Insertion point: CONFIRMED.** Insert at the **parse→sheets boundary**, in a **shared helper** called by both the process-job worker and the analyze path. Only point where the raw `{header:1}` grid exists, and it precedes the fingerprint (which must see the corrected header). The shared helper reinforces the singular path — one de-band, two callers, identical behavior, invoked at the same point — it is not a parallel path.

**D2 — Scale composition: ADJUSTED (split by data-locality, not by sheet).**
- *Header recovery* (§3.2a–2b) needs only the top ~8 rows, which the OB-251 window always materializes. It runs on the windowed sample for **every** sheet, fixing the `__EMPTY` defect and the fingerprint **universally and scale-safely** — large extracts with a banner row included.
- *Full-grid operations* (§3.2c–2f: section lift, carry-down, subtotal/blank removal, narrative capture) run only when the full grid materializes (within the OB-251 cell ceiling). Over-ceiling sheets receive header recovery + a degenerate single-block body.
- *No-silent-fallback safeguard:* an over-ceiling sheet whose windowed sample exhibits banded structure (section-label or subtotal rows in the sample) emits a structured observation flagging "banded structure exceeding de-band ceiling," never a silent degenerate pass.

SR-2 is honored on structural grounds: a sheet large enough to exceed the cell ceiling is, by structure class, a machine-generated uniform extract — it does not carry the embedded sections / carry-down / subtotals that full-grid de-banding exists to handle. A human-authored banded report is small by nature (Casa Diaz: 257 rows). One helper; grid-availability is an input property handled within the transform, not a structure-class branch (HALT-4 not triggered).

**D3 — Commit reproducibility: CONFIRMED.** Carry the **per-row transform map** from the parse-boundary stage into the windowed-commit (`windowed-commit.ts`); never re-derive the de-band at commit. The map, keyed by source row index, carries: per-row disposition (record vs sidecar, with reason), the `__section` value, the carried-down identity values, and the raw-column-index → tidy-column-name mapping (with dup suffixes), plus the documentation unit's row set. The windowed-commit **applies** the map by index; it does not recompute it (preserves PG-11 byte-identical reproducibility — the commit is deterministic given the map).
