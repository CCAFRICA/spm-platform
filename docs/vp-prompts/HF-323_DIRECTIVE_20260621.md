# HF-323 -- Remove Import-Time Count-Metric Discriminator

## S0 -- CC Standing Rules Header

Binds `CC_STANDING_ARCHITECTURE_RULES.md` v2.0 throughout.

Branch: `hf-323-remove-import-discriminator`

---

## S1 -- Objective

HF-322 placed a count-metric discriminator at import time in `plan-orchestration.ts`, between `const ci = ...` and `constructTree(ci)`. This is architecturally wrong. Import is pure plan comprehension: the LLM reads the plan, emits the compositional intent, the constructor builds the DAG. No data inspection, no override.

The discriminator requires period and convergence context that does not exist at import time. It failed in production because periods are created after plan import.

**This HF is subtraction only.** Remove the discriminator call from `plan-orchestration.ts`. Remove the import statement. The discriminator file and tests may remain in the tree (they are not harmful as unused code; removing them is optional). No replacement is added in this HF.

---

## S2 -- Constraints

**C1 -- SUBTRACTION ONLY.** No new code. No new logic. No calc-time discriminator. No replacement mechanism.

**C2 -- IMPORT RESTORED TO PURE PLAN COMPREHENSION.** After this HF, plan import emits whatever the LLM emits, and the constructor builds it. No interception, no mutation of the compositional intent.
