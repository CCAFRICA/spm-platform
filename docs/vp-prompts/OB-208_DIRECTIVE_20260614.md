## §1 — PROBLEM STATEMENT

### 1.1 Three passes shipped; the visualization paradigm never arrived

OB-205/HF-291/OB-206/OB-207 corrected data wiring, routing, regime classification, and the agent-nav spine. None delivered the visualization paradigm DS-003 specifies. The live Decide/Intelligence (/stream) and Decide/Results (/operate/results) surfaces use ~two component types (HeroMetric, HorizontalBar) where DS-003 mandates the full nine-component library with per-surface assignments. The result, in the architect's words: "no real information being displayed, no AI leverage front and center, Bloodwork not showing, no diversity of visualization surfaces."

### 1.2 The five defects, each answerable from substrate (not new design)

**D-1 — Field-identity hardcoding blocks regime-3 attainment (Korean Test violation).** The engine emits `meta_colocacion` in the DAG grammar and persists `Meta_Colocacion` in metrics — two independent hardcoded strings for one field. The Korean Test (T1-E910 v2) requires one canonical declaration from which every boundary derives. Because field identity has two sources, the regime-3 attainment ratio (actual ÷ target) is uncomputable without name-matching (which the Korean Test forbids), so Colocación/Captación show "target-driven" but not the number. This is the import-layer Carry-Everything/Korean-Test violation reappearing at the calculation layer.

**D-2 — No drill-down (DS-003 Rule 6 violated).** DS-003 §Rule 6: "Primary surface is glanceable (Type 1). Click/hover/expand reveals detail (Type 2)." Every claim the surface makes ("4 entities >2σ above mean," "3 entities have identical payout of $372," every distribution bar, every component bar) MUST be a click target that reveals the underlying entities and lets the screen verify its own assertion. Currently claims are dead text. A claim the user cannot drill into to verify is an unverifiable assertion.

**D-3 — Drill-through is unintelligent (would be a glorified table).** When a claim is drilled, the revealed view must be a further Five-Elements synthesis (DS-013) — summarized, contextualized, actionable — not a raw entity table. Progressive disclosure deepens intelligence; it does not dump rows.

**D-4 — Static defaults instead of adaptive (Synaptic signal capture unused).** Surfaces should default expanded; capture the user's collapse as a `stream_interaction` signal (classification_signals, per the Synaptic State spec); if the user repeatedly collapses a surface, the persisted preference flips its default to collapsed. The UI's own defaults act on captured signal (Thermostat principle applied to the interface).

**D-5 — Visualization monoculture (DS-003 Diversity Minimum + component library violated).** DS-003 §Rule 1: any page with 4+ data elements uses 3+ distinct component types or it has failed Cognitive Fit. Both Decide surfaces fail this. The nine-component library (HeroMetric, GaugeMetric, HorizontalBar, StackedBar, NeighborhoodLeaderboard, DistributionPosition, Sparkline, ThresholdArea, PrioritySortedList, ConfigurablePipeline, SteppedProgress, RadarChart, FlowDiagram) is built once and the surfaces compose from it per DS-003 §Section 4's per-surface assignments. The Insight Agent narrative (DS-013) leads each surface — AI front and center.

**D-6 — Dead Simulate control (Action Proximity).** "Simulate" on /stream Optimization Opportunities no-ops. It must open the inline impact model (DS-008-A3 sensitivity) — the same class as the HF-292/CLT-167 dead-control discipline.

### 1.3 Defect class

DS-003 (a ratified, implementable visualization specification) was never built — surfaces shipped with a two-component monoculture, no progressive disclosure, no adaptive defaults, and no AI narrative. Plus a Korean Test violation at the field-identity layer (D-1) and a dead control (D-6). All answerable from existing substrate.

## §5 — PHASE 3: DRILL-DOWN + ADAPTIVE DEFAULTS (D-2, D-3, D-4)

### 5.1 The drill-through intelligence view (D-2, D-3)

A reusable drill-through surface: given a claim/aggregate target (e.g., "the 4 entities >2σ," "the entities in this distribution band," "this component's contributors"), it renders a **Five-Elements synthesis** of those entities — NOT a raw table:
- value (the subset's aggregate), context (how this subset compares to the population), comparison (subset vs mean/peer), action (what to do about them — Investigate/Coach/Resolve as applicable), impact (the cost/opportunity of this subset).
- The entities ARE listed (so the claim is verifiable — the user sees the 4 entities and counts them), but framed by the synthesis above them, with per-entity context (each entity's relevant metric, not every column).

Every claim on both Decide surfaces wires its `onDrillDown` to this view. **The verification principle:** "3 entities have identical payout of $372" → click → see exactly those 3 entities, each showing $372, so the screen verifies its own assertion. SR-38: the drill-through's aggregate matches the claim (the count and the figure reconcile).

**HALT-2:** a claim's underlying entity set cannot be resolved from persisted data (the claim is computed but the contributing entities aren't recoverable). Report which claim; the claim either becomes drill-disabled with a noted reason or the computation is adjusted to retain the entity set.

### 5.2 Adaptive defaults via signal capture (D-4)

- Surfaces (and collapsible sections within them) default **expanded**.
- On collapse/expand, write a `stream_interaction` signal to `classification_signals` (per the Synaptic State spec — batched, signal_value JSONB: {surface, section, action, persona}). FP-49: confirm classification_signals columns before writing.
- On surface load, read the persisted interaction preference; if the user has repeatedly collapsed a section (a threshold, e.g. ≥N of last M loads), default it collapsed. The default is data-driven (Thermostat).
- **Korean Test:** the signal schema is structural (surface/section ids, not domain terms).

**HALT-3:** classification_signals cannot accept stream_interaction writes (RLS, or the signal_type enum doesn't permit it). Report; the adaptive default degrades to a session-scoped preference (not persisted) with the persistence noted as a residual.
