## §1 — Problem Statement

### 1.1 — Why this audit exists
The import → plan-interpretation → save path has been investigated **piecemeal** across DIAG-057, AUD-0014, HF-255/256/257, DIAG-058, and an open duplicate-execution question — each read scoped to the failure in front of it, each discovering one more surface late. This AUD produces the **missing companion**: the complete, SHA-pinned, read-only execution trace for the ingestion → plan-interpretation half, to the same standard, reconciled with AUD-005 at the calc boundary.

### 1.2 — What "single source of truth" means here (the deliverable's standing)

### 1.3 — Scope (the complete spine, end to end)
From the user clicking import to the rule_set persisted and signals emitted, INCLUDING the cross-cutting paths that piecemeal reads have missed. The trace must capture, for the live HEAD:
1. **Ingestion entry & transport** — upload UI, the per-file route decision, analyze vs analyze-document, proposal assembly, what is threaded to execute (storagePaths map, documentMetadata incl. `fileBase64`).
2. **Execute-bulk dispatch hub** — request shape, per-file download loop, format-aware parse (document-skip vs workbook), the classification switch (entity/target/transaction/reference) and the four data pipelines, the plan grouping + the sole batched plan dispatch (post HF-257), commit.

4. **Re-entry / resume / retry / async** — the `reimport-resume` path, batch lifecycle, anything that can re-drive a batch, the sync-analyze vs async `process-job` classifier split.
5. **Cross-cutting** — fingerprint/moat tiers + `fieldBindings` injection (HF-254); classification_signals + comprehension emission (Decision 64/153); transport (Storage vs base64, AP-1); per-stage schema touchpoints (tables read/written).
