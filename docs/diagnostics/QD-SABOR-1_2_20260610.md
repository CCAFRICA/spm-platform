## QD-2 ‚Äî login-path session lifecycle (the kill)

**1. `auth-context.tsx:237-275` login flow:** `signInWithEmail` (:240) ‚Üí [no MFA branch in client flow] ‚Üí `fetchCurrentProfile` (:254) ‚Üí `if (!profile)` (:255) ‚Üí `'Account found but profile is missing. Contact your administrator.'` (:256).

**2. `signInWithEmail` (auth-service.ts:32-48):** returns `signInWithPassword` data; **no AAL/factor check, no signOut, no MFA logic**; awaits `logAuthEventClient('auth.login.success')` (:46) ‚Üí `POST /api/auth/log-event`, which **traverses middleware** before `fetchCurrentProfile`. MFA vs non-MFA diverge only server-side later (middleware.ts:265-285), not in this flow.

**3. Idle-expiry ‚Äî server-side authoritative (`middleware.ts`):**
- reads `vialuce-last-activity` (:204); compares `(now - lastActivityMs) > IDLE_TIMEOUT_MS` (30m, :241);
- on expiry: `clearAuthCookies(request, idleResponse)` (:244) zeroes every `sb-*` cookie (:71-82) + clears session-start/last-activity (:245-246); redirect `/login?reason=idle_timeout`.
- cookie `vialuce-last-activity` is **session-scoped (no maxAge)**, initialized only if absent (:215-221), refreshed only `if (existingLastActivity)` (:250-258).
- middleware runs on **every** request incl. the awaited `/api/auth/log-event` POST ‚Äî i.e. between password success and `fetchCurrentProfile`.
- (client timer `SessionExpiryWarning.tsx:42-56` also emits the event but only `window.location.replace`, never clears `sb-*`.)

**4. Non-MFA sequence + the kill line (3 lines):**
1. `auth-context.tsx:240` sets `sb-*` ‚Üí `:46` awaited log POST traverses middleware ‚Üí `:254` `fetchCurrentProfile` ‚Üí `auth-service.ts:182` `getUser()` ‚Üí `resolve-identity.ts:80` query.
2. **Session-destruction line `middleware.ts:244` `clearAuthCookies`** (idle; `:234` absolute) ‚Äî if a stale `vialuce-last-activity` from a prior session (>30m, read :204/:241) rides along, it zeroes the just-set `sb-*` on a response the browser applies.
3. Consumed at `auth-service.ts:182-183` (`getUser()` ‚Üí `return null`) ‚Üí short-circuits before `resolve-identity.ts:80` ‚Üí `auth-context.tsx:256` "profile is missing" (mislabeled ‚Äî session was absent, not the profile).
