## 1. The paradigm, in brief (so the new Claude holds it correctly)

The new Claude must hold these or it will drift. Each is developed fully in DS-031; this is the compression.

- **The platform is one hierarchical generative model that minimizes surprise about its inputs.** Comprehension (Thalamus), calculation, and convergence are *levels of one model*, not separate pipelines.
- **Structure composes upward** (atom/nanosignal → fingerprint → comprehension), **prediction flows downward** (accumulated signal sets expectation at every level), **only prediction error propagates upward** (the unrecognized residue), and **consolidation writes resolutions back** (today's surprise becomes tomorrow's silent prediction — the flywheel).
- **The discrete-agent decomposition is dissolved.** "Normalize / reconcile / dedup / anomaly" are not operations performed in sequence; they are *names for kinds of pattern the surface recognizes.* The right answer requires **all the analysis co-present and mutually informing** — sequencing destroys the joint information. There is one recognizing surface; the operations are facets of one recognition. **This supersedes the EECI five-agent roadmap.**
- **The apex (Decision 158, sharpened):** when prediction error reaches the top — genuine novelty no level can predict — a single bounded LLM expression extends the model, then consolidates so that structure is predicted (free) forever after. The LLM expresses at the apex on irreducible surprise; it does not drive agents. Cost decreases with usage, mechanically.
- **Recall makes co-presence affordable.** Joint analysis is expensive (why the frontier must approximate or sequence). The accumulated surface collapses it: most structure is *recognized*, so expensive joint reasoning runs only over the small residue at the apex. **The unit of learning is the network, not the dataset** — that is the structural moat.
- **Precision-weighting is the trust safeguard** (the one genuinely new specification in DS-031). A confident predictive model *hallucinates* — predicts away real signal it should surface. For a trust platform that is the worst failure mode. Precision-weighting overrides the efficiency path toward *surfacing* on high-consequence / thin-exposure structure, so trust-critical surprise is never silently predicted away. **This is what makes it a trust paradigm, not merely an efficient one.**
- **"Not agentic" is correct, not a limitation.** The product is *justified trust*; trust demands the most deterministic, auditable, reconcilable system — the opposite of the agentic frontier. The architecture of trustable AI is recognition + bounded apex expression + deterministic construction + precision-weighting. Capability disrupts the market; vocabulary does not.

## 7. Named anti-patterns to avoid (from the originating session)

- **Scope narrowing disguised as methodology.** In the originating session, a "verify-first" directive (HF-353) silently converted a concrete build requirement ("deliver working access") into an investigation ("check whether access exists"), and merged having only investigated — the access still did not exist. **Verification is a step inside a build, never a substitute for the deliverable.** When the new Claude writes directives, the deliverable must be stated as a guarantee, with an explicit anti-narrowing clause.
- **PRISM-centering.** An early draft of the paradigm document over-weighted PRISM as the *origin* of the paradigm. Correct framing: **Vialuce is the product; the four principles are the foundation (present-tense, already governing); Thalamus/PRISM is one expression among the engine, convergence, and experience.** Do not let the layer being built become the center of the platform's story.
- **Calling a transform "agentic."** The Normalizer is a learned transform with one bounded LLM call, not an autonomous agent — and that is Decision 158 working as designed. Do not borrow "agentic" for hype; reserve it for genuine orchestration autonomy (which DS-031 deliberately does not build).
- **Citation-without-application / favorable-framing drift / fabrication of state.** Cite a principle then violate it in output; rationalize a regression as the system "getting more correct"; assert a file path / column / capability state without verification. All three appear in the project's persistent defect registry.
